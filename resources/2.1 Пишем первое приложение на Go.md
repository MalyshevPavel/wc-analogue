Напишем аналог утилиты wc в Linux. Из `man wc`:
```bash
NAME
     wc – word, line, character, and byte count

DESCRIPTION
     The wc utility displays the number of lines, words, and bytes contained in each input file, or standard input (if no file is specified) to the standard output.  A line is defined as a string of characters delimited by a ⟨newline⟩ character.  Characters beyond the final ⟨newline⟩ character will not be included in the line count. 
```

Работает она следующим образом

```bash
george@MacBook-Pro-Georgij ~ % echo "one two three\nfour five" | wc
 2       5      24
george@MacBook-Pro-Georgij ~ % echo "one two three\nfour five" > a.txt
george@MacBook-Pro-Georgij ~ % wc a.txt
2       5      24 a.txt
```


То есть у утилиты есть два режима работы - когда она читает из файла и из stdin. В stdout она выводит кол-во слов, строк и байтов прочитанных из источника. Если читали из файла - еще выводится файл.

### Как пишем

Для написания программы создадим директорию с проектом и вызовем внутри нее `go mod init <project_name>`.

- Создадим файл main.go, в котором реализуем логику приложения. 
- В этом файле создаем точку входа в приложение - `func main() {...}`, не забывая указать имя пакета `package main`
- Для чтения аргументов командной строки используем `os.Args`
- Для открытия файла можно использовать `os.Open
- Для чтения из стандартного потока ввода - `os.Stdin`
- Разбор потока данных можно сделать с помощью библиотеки буфферизированного чтения `bufio`. *Подсказка*: обращаем внимание, что `os.File` имплементирует интерфейс `io.Reader`. `os.Stdin` и файл, полученный из `os.Open` являются `os.File`
- Для проверки является ли символ пробельным можно использовать `unicode.IsSpace()`

При помощи этих конструкций реализуем алгоритм. Полезно потренироваться составить его самостоятельно. Концептуально: нужно прочитать весь входной поток байтов (из stdin или заданного файла) и посчитать кол-во кол-во слов (последовательность символов, ограниченных пробельным символом), кол-во строк (символ перевода строки - '\n') и кол-во байт. Полученный результат нужно вывести в stdout в заданном формате.

После того как реализовали приложения - собираем с помощью команды

```bash
go build -o wordcounter .
```


И запускаем приложение

```bash
echo "one two three\nfour five\n" | ./wordcounter`
```  

Проверяем, что работает аналогично wc. Не забываем про кейс с чтением из файла.

### Тесты

Любое приложение нужно тестировать. Создадим файл main_test.go и напишем юнит-тесты с помощью пакета testing. Можно посмотреть небольшой видео про юнит тестирование https://www.youtube.com/watch?v=AprUuq_rm0I
Обрати внимание, как в конце видео все кейсы организованы в массив.
Придумываем и описываем основные тест кейсы аналогичным образом.

Обращаем внимание, что функцию main протестировать не получится т.к. нет возможности ее вызвать из файла с тестами. Подсказка: у нас в приложении есть логика получения входных данных(внешний интерфейс получения входных данных), логика вычисления(бизнес логика) и логика отображения ответа(внешний интерфейс выдачи результата). Их лучше разделить, тогда код будет протестировать легче.
